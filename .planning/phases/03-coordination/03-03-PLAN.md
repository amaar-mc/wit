Contract system: propose/respond RPC, signature extraction, git pre-commit hook for enforcement.

Purpose: Agents working on shared interfaces need to agree on function signatures and type shapes. This plan lets one agent propose a contract (e.g. "validateToken must have signature (token: string): boolean"), another agent accepts it, and a git pre-commit hook prevents commits that violate accepted contracts.
Output: `contracts` table, `contract.propose`/`contract.respond`/`contract.query`/`check-contracts` RPC methods, `wit hook install` CLI command, pre-commit hook shell script.

  <action>
1. Create `src/cli/commands/hook.ts` with a clipanion `HookInstallCommand`:
   - Command path: `hook install`
   - Resolves repo root via process.cwd() (wit is always run from within a repo)
   - Gets git hooks directory: use `git rev-parse --git-path hooks` (via Bun.$) to respect custom core.hooksPath, per research pitfall 4. Fall back to `.git/hooks` if git command fails.
   - Creates hooks directory if it doesn't exist (mkdirSync recursive)
   - Writes pre-commit hook shell script to `{hooksDir}/pre-commit`:
     ```sh
     #!/bin/sh
     # Managed by wit. Do not edit -- run `wit hook install` to regenerate.
     STAGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(ts|tsx|py)$')
     if [ -z "$STAGED_FILES" ]; then exit 0; fi
     REPO_ROOT=$(git rev-parse --show-toplevel)
     echo "$STAGED_FILES" | bun run --cwd "$REPO_ROOT" wit check-contracts
     exit $?
     ```
   - Sets chmod 0o755 on the hook file
   - The `wit check-contracts` CLI command reads file paths from stdin, gets staged content via `git show :filepath`, calls the `check-contracts` RPC method on the daemon, and exits 0/1 based on violations.
   - Prints "Pre-commit hook installed at {hookPath}" on success

2. Add a `CheckContractsCommand` to `src/cli/commands/hook.ts` (or a sibling file):
   - Command path: `check-contracts`
   - Reads file paths from argv (remaining args after command name)
   - For each file, reads staged content via `Bun.$\`git show :${filePath}\``
   - Calls the daemon RPC `check-contracts` with `{files: [{path, content}, ...]}` using the existing `rpc()` client from `src/cli/client.ts`
   - If violations returned, prints each violation and exits with code 1
   - If no violations, exits 0
   - If daemon is unreachable after 2 seconds, exits 0 (best-effort enforcement per research recommendation)

3. Register both commands in `src/cli/index.ts`:
