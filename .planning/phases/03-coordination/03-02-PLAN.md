    - File-level intent (no symbols, null byte range) overlaps with any intent on same file
    - Agent declares intent on symbol locked by another session -> LOCK_INTERSECTION conflict item with symbolPath, heldBy, expiresAt
    - Agent declares intent on symbol whose callee is locked by another session -> DEP_CHAIN conflict item with intentSymbol, lockedCallee, heldBy
    - ConflictReport always has {hasConflicts: boolean, items: ConflictItem[]} shape
    - No conflicts returns {hasConflicts: false, items: []}
    - intent.declare response changes from {intentId} to {intentId, conflicts: ConflictReport}
  </behavior>
  <action>
1. Add types to `src/shared/protocol.ts`:
```typescript
export type ConflictItem =
  | { type: "INTENT_OVERLAP"; overlappingIntentId: string; overlappingSessionId: string; description: string; }
  | { type: "LOCK_INTERSECTION"; symbolPath: string; heldBy: string; expiresAt: string; }
  | { type: "DEP_CHAIN"; intentSymbol: string; lockedCallee: string; heldBy: string; };

export type ConflictReport = { hasConflicts: boolean; items: ConflictItem[]; };
```

2. Add three conflict detection functions to `src/daemon/rpc/handlers.ts`:

   **findOverlappingIntents(deps, sessionId, files, startByte, endByte):**
   - Query intents table for other sessions with status in ["declared","active"]
   - If startByte/endByte are not null, use byte-range overlap predicate: `intents.startByte < endByte AND intents.endByte > startByte` (add `IS NOT NULL` guard on stored intents)
   - Also check file-level overlap: for intents where the stored intent has null byte range OR the new intent has null byte range, match on file containment using LIKE on comma-delimited files column (for each file in the declaring intent's files array, check `intents.files LIKE '%,file,%'`)
   - Return array of INTENT_OVERLAP ConflictItems

   **findLockConflicts(deps, sessionId, symbolPaths):**
   - For each symbol path, check if an active lock exists held by a different session (eq on symbolPath, gt on expiresAt vs now, ne on sessionId)
   - Return array of LOCK_INTERSECTION ConflictItems
   - If symbolPaths is empty, return []

   **findDepChainConflicts(deps, sessionId, symbolPaths):**
   - For each symbol path, query symbolDeps where caller = symbolPath to find callees
   - For each callee, check if it has an active lock held by a different session
   - Return array of DEP_CHAIN ConflictItems
   - This mirrors the existing buildCallerWarnings pattern but inverted: checking callees instead of callers

3. Add **buildConflictReport(deps, {id, sessionId, files, symbols, startByte, endByte}):**
   - Call all three finders, concatenate results
   - Return `{hasConflicts: items.length > 0, items}`

4. Modify `intent.declare` case in handleRpc:
   - After inserting the intent row (already done in Plan 01), call buildConflictReport
   - Change return from `{intentId: id}` to `{intentId: id, conflicts: conflictReport}`
   - The intent ALWAYS succeeds (conflicts are warnings, never blocks) per research anti-patterns

5. Write tests:
   - Two sessions declare intents on overlapping byte range -> INTENT_OVERLAP in conflicts
   - Two sessions declare intents on non-overlapping ranges -> no conflicts
   - Session declares intent on symbol that another session has locked -> LOCK_INTERSECTION
   - Session declares intent on symbol whose callee is locked -> DEP_CHAIN
   - Session declares intent with no conflicts -> {hasConflicts: false, items: []}
   - File-level intent (no symbols) on same file as another intent -> INTENT_OVERLAP

   Tests requiring locks: use the existing lock.acquire handler to set up locks before declaring intents.
