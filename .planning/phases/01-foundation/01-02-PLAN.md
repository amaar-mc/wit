Build the Hono app and RPC handler layer.

`src/daemon/server.ts`:
- Create Hono app instance
- Add middleware on `/rpc` that:
  1. Parses body with `c.req.json()` -- if parsing fails, return -32700 PARSE_ERROR
  2. Validates `witVersion` field matches PROTOCOL_VERSION -- if not, return -32001 VERSION_MISMATCH
  3. Validates basic JSON-RPC shape (jsonrpc="2.0", id exists, method is string) -- if not, return -32600 INVALID_REQUEST
  4. Stashes parsed body in context via `c.set("rpcBody", body)` -- this is critical to avoid the double-parse pitfall from research
  5. Calls `next()`
- Add POST `/rpc` route that reads body from `c.get("rpcBody")` and calls `handleRpc(body, deps)`
- The `deps` object is a dependency injection bag: `{ db, sqlite }` -- passed from the daemon index at startup via `app.use("*", async (c, next) => { c.set("deps", deps); await next(); })`
- Export a `createApp(deps: DaemonDeps)` factory function (not a bare `app` singleton) so tests can inject mock deps
- Define `DaemonDeps` type: `{ db: BunSQLiteDatabase; sqlite: Database }`

`src/daemon/rpc/handlers.ts`:
- Export `handleRpc(body: RpcRequest, deps: DaemonDeps): RpcSuccess | RpcError`
- Switch on `body.method`:
  - "ping": return `createRpcSuccess(body.id, "pong")`
  - "register": validate params has `name` (string) and `sessionId` (string) using zod, insert into agents table via `deps.db.insert(agents).values(...)`, return `createRpcSuccess(body.id, { agentId: result.lastInsertRowid })`
  - default: return `createRpcError(body.id, -32601, "METHOD_NOT_FOUND")`

Build the daemon process lifecycle and entry point.

`src/daemon/lifecycle.ts`:
- Export `writePidFile(pidPath: string): Promise<void>` -- writes `String(process.pid)` to pidPath via `Bun.write`
- Export `cleanStaleSocket(socketPath: string): void` -- wraps `unlinkSync(socketPath)` in try/catch (no-op if file missing). This runs BEFORE Bun.serve to avoid EADDRINUSE from a crashed daemon leaving a stale socket.
- Export `setupShutdownHandlers(ctx: { sqlite: Database; pidPath: string; socketPath: string; server: Server }): void`:
  - Register handlers for both SIGTERM and SIGINT
  - Handler: `server.stop()`, `sqlite.close()`, `unlinkSync(pidPath)` (try/catch), `unlinkSync(socketPath)` (try/catch), `process.exit(0)`
  - Import `unlinkSync` from `node:fs`

`src/daemon/index.ts`:
- Import `createDatabase` from `../db/index`
- Import `runMigrations` from `../db/migrate`
- Import `createApp` from `./server`
- Import `writePidFile`, `cleanStaleSocket`, `setupShutdownHandlers` from `./lifecycle`
- Import `SOCKET_PATH`, `PID_PATH`, `DB_PATH`, `WIT_DIR` from `../shared/paths`
- At top level (this file runs as the daemon process):
  1. `mkdirSync(WIT_DIR, { recursive: true })` -- ensure .wit/ exists
  2. `const { db, sqlite } = createDatabase(DB_PATH)`
  3. `await runMigrations(db)`
  4. `cleanStaleSocket(SOCKET_PATH)` -- prevent EADDRINUSE
  5. `await writePidFile(PID_PATH)`
  6. `const app = createApp({ db, sqlite })`
  7. `const server = Bun.serve({ unix: SOCKET_PATH, fetch: app.fetch })`
  8. `setupShutdownHandlers({ sqlite, pidPath: PID_PATH, socketPath: SOCKET_PATH, server })`
  9. Log "wit daemon listening on {SOCKET_PATH}"
