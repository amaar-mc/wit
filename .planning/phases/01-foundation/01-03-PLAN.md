Build the CLI-to-daemon communication layer. This is the "connect-or-spawn" pattern -- the central design idiom of Phase 1.

`src/cli/client.ts`:

Export `isDaemonAlive(): Promise<boolean>`:
- Check if PID_PATH exists via `Bun.file(PID_PATH).exists()`
- If no PID file, return false
- Read PID from file, parse as int
- Call `process.kill(pid, 0)` in try/catch -- if throws, process is dead
- If dead: delete stale PID file (`unlinkSync(PID_PATH)` in try/catch), return false
- If alive: return true

Export `spawnDaemon(): Promise<void>`:
- `Bun.spawn(["bun", "run", "src/daemon/index.ts"], { detached: true, stdin: "ignore", stdout: "ignore", stderr: "ignore", env: { ...process.env, WIT_REPO_ROOT: process.cwd() } })`
- Call `proc.unref()` immediately -- parent CLI must not wait for child
- CRITICAL: pass `WIT_REPO_ROOT` env var so daemon knows which repo it serves. This resolves the open question from research about CWD loss in detached processes.

Export `waitForSocket(timeoutMs: number): Promise<void>`:
- Poll `Bun.file(SOCKET_PATH).exists()` every 50ms
- If socket appears, return
- If timeout reached, throw `Error("Daemon did not start within {timeoutMs}ms")`
- Use `Bun.sleep(50)` between polls to avoid starving the event loop

Export `ensureDaemon(): Promise<void>`:
- If `isDaemonAlive()` returns true, return immediately
- Otherwise: call `spawnDaemon()`, then `waitForSocket(3000)`

Export `rpc<T>(method: string, params: unknown): Promise<T>`:
- Call `ensureDaemon()` first
- Use `fetch("http://localhost/rpc", { unix: SOCKET_PATH, method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify(createRpcRequest(method, params)) })`
- Parse response JSON
- If `body.error` exists, throw `Error("RPC error [${body.error.code}]: ${body.error.message}")`
- Return `body.result as T`

Build the CLI entry point and the init command -- the first user-facing feature of Wit.

`src/cli/commands/init.ts`:
- Import `Command` from `clipanion`
- Export `InitCommand` class extending `Command`:
  - `static paths = [["init"]]`
  - `static usage = Command.Usage({ description: "Initialize wit in the current repository" })`
  - `async execute(): Promise<number>`:
    1. `mkdirSync(WIT_DIR, { recursive: true })` -- create .wit/ directory
    2. `const { db, sqlite } = createDatabase(DB_PATH)` -- open/create SQLite
    3. `await runMigrations(db)` -- apply schema
    4. `sqlite.close()` -- close the CLI's connection (daemon will open its own)
    5. `await ensureDaemon()` -- start daemon if not running
    6. `this.context.stdout.write("Wit initialized.\n")`
    7. Return 0

Note: The init command opens the DB to run migrations, then closes it. The daemon opens its own connection independently. This avoids sharing a Database instance across processes.
