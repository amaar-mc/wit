  <action>
Build the database layer. This is the persistence foundation for the entire project.

`src/db/schema.ts`:
- Define `agents` table using drizzle-orm/sqlite-core: `sqliteTable("agents", { ... })`
- Columns: `id` (int, primaryKey, autoIncrement), `name` (text, notNull), `sessionId` (text("session_id"), notNull, unique), `connectedAt` (int("connected_at", { mode: "timestamp" }), notNull)
- Export the table definition

`src/db/index.ts`:
- Export a `createDatabase(dbPath: string)` function (NOT a module-level singleton) that:
  1. Creates `new Database(dbPath, { create: true })` from `bun:sqlite`
  2. Runs all four PRAGMAs in order: journal_mode=WAL, busy_timeout=5000, synchronous=NORMAL, foreign_keys=ON
  3. Wraps with `drizzle({ client: sqlite, schema })`
  4. Returns `{ db, sqlite }` (db = Drizzle instance, sqlite = raw Database for shutdown close)
- Using a factory function instead of a singleton avoids the "daemon migration runs against wrong DB path" pitfall from research

`src/db/migrate.ts`:
- Export `runMigrations(db: BunSQLiteDatabase)` that calls `migrate(db, { migrationsFolder: "./drizzle" })`
- Import migrate from `drizzle-orm/bun-sqlite/migrator`

After creating source files, generate the initial migration:
- Run `bunx drizzle-kit generate` to produce the SQL migration files in `./drizzle/`

`src/db/db.test.ts`:
- Test that createDatabase sets WAL mode: open temp db, check `PRAGMA journal_mode` returns "wal"
- Test that busy_timeout is 5000: check `PRAGMA busy_timeout`
- Test that synchronous is NORMAL (1): check `PRAGMA synchronous`
- Test that foreign_keys is ON (1): check `PRAGMA foreign_keys`
- Test that agents table exists after migration: run migrations, query sqlite_master for "agents" table
- Use a temp directory for test databases, clean up after each test
