    - symbols.test PY: decorated function still extracted correctly (decorator does not break name capture)
  </behavior>
  <action>
    1. Install dependencies: `bun add web-tree-sitter@0.24.7 tree-sitter-wasms`

    2. Create `src/parser/loader.ts`:
       - Export `ParserService` type: `{ typescript: Parser.Language; python: Parser.Language; parser: Parser }`
       - Export `createParserService(wasmDir: string, treeSitterWasm: string): Promise<ParserService>`
       - Call `Parser.init({ locateFile: () => treeSitterWasm })` once
       - Load TS WASM: `await Parser.Language.load(await Bun.file(join(wasmDir, "tree-sitter-typescript.wasm")).bytes())`
       - Load Python WASM: `await Parser.Language.load(await Bun.file(join(wasmDir, "tree-sitter-python.wasm")).bytes())`
       - Create `new Parser()` and return the service object
       - Export a `defaultWasmPaths()` helper that resolves paths relative to `import.meta.url` into `node_modules/` -- same CWD-independence pattern as migrate.ts

    3. Create `src/parser/symbols.ts`:
       - Export `SymbolInfo` type: `{ name: string; kind: "function" | "method" | "type" | "interface" | "class" | "arrow"; startLine: number; endLine: number; startByte: number; endByte: number }`
       - Export `extractSymbols(parser: Parser, language: Parser.Language, source: string): SymbolInfo[]`
       - Use two query strings: `TS_SYMBOL_QUERY` and `PY_SYMBOL_QUERY` (constants, not exported)
       - TS query must include: function_declaration, variable_declarator+arrow_function (Pitfall 4 from research), method_definition, type_alias_declaration, interface_declaration, class_declaration
       - PY query must include: function_definition, class_definition
       - Use `language.query(queryString).matches(tree.rootNode)` to get matches
       - Map each match: find "name" capture for symbol name, find "definition.*" capture for byte/line range
       - Determine `kind` from the capture name suffix (definition.function -> "function", definition.arrow -> "arrow", etc.)
       - Sort results by startLine ascending
       - IMPORTANT: Between `parser.setLanguage(language)` and `parser.parse(source)`, do NOT await anything (parser is not thread-safe per research anti-patterns)

    4. Write tests RED first, then implement GREEN.
       - In loader.test.ts: call `createParserService(defaultWasmPaths().wasmDir, defaultWasmPaths().treeSitterWasm)` and assert the three properties exist. Parse a trivial TS string and Python string to confirm languages load.
       - In symbols.test.ts: create a `beforeAll` that initializes the parser service once. Each test provides a source string and asserts extractSymbols returns the expected SymbolInfo array.

  <name>Task 2: Add locks and symbol_deps tables to Drizzle schema and generate migration</name>
  <files>src/db/schema.ts</files>
  <action>
    1. Edit `src/db/schema.ts` -- add `locks` and `symbolDeps` tables alongside existing `agents` table:
       - Add imports: `index`, `uniqueIndex` from "drizzle-orm/sqlite-core"
       - `locks` table:
         - `id: int("id").primaryKey({ autoIncrement: true })`
         - `symbolPath: text("symbol_path").notNull()` -- format: "src/auth.ts:validateToken"
         - `sessionId: text("session_id").notNull()` -- references agents.sessionId logically (not FK -- agent may disconnect before lock expires via TTL)
         - `acquiredAt: int("acquired_at", { mode: "timestamp_ms" }).notNull()` -- timestamp_ms for numeric comparison in TTL cleanup
         - `expiresAt: int("expires_at", { mode: "timestamp_ms" }).notNull()` -- TTL expiry
         - Index: `uniqueIndex("locks_symbol_path_unique").on(t.symbolPath)` -- one lock per symbol
       - `symbolDeps` table:
         - `id: int("id").primaryKey({ autoIncrement: true })`
         - `file: text("file").notNull()` -- relative path: "src/auth.ts"
         - `caller: text("caller").notNull()` -- qualified: "src/auth.ts:login"
         - `callee: text("callee").notNull()` -- qualified: "src/auth.ts:validateToken" or "?:unknownFn" for cross-file
         - Indexes: `index("symbol_deps_callee_idx").on(t.callee)`, `index("symbol_deps_caller_idx").on(t.caller)`, `index("symbol_deps_file_idx").on(t.file)`

    2. Generate migration: `bunx drizzle-kit generate`
       - This creates a new SQL file in `drizzle/` with CREATE TABLE for locks and symbol_deps
       - Verify the generated SQL contains both tables and all indexes

    3. Run existing test suite to confirm nothing is broken: `bun test`
