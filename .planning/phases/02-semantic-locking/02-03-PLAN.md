<task type="auto" tdd="true">
  <name>Task 1: Call edge extraction for TypeScript and Python with tests</name>
  <files>src/parser/calls.ts, src/parser/calls.test.ts</files>
  <behavior>
    - TS: function A calls function B in same file -> edge { caller: "A", callee: "B" } returned
    - TS: method inside class calls standalone function -> edge { caller: "methodName", callee: "fnName" }
    - TS: arrow function calls another function -> edge { caller: "arrowName", callee: "target" } (caller name from variable_declarator parent)
    - TS: member expression call (obj.method()) -> callee is "method" (property_identifier text)
    - TS: call at module level (not inside any function) -> no edge produced (no containing function)
    - TS: nested function calls -> edge caller is the innermost containing function
    - PY: function a calls function b -> edge { caller: "a", callee: "b" }
    - PY: method inside class calls function -> edge { caller: "method", callee: "fn" }
    - PY: attribute call (obj.method()) -> callee is "method"
    - PY: call at module level -> no edge produced
    - qualifyEdges with filePath "src/auth.ts" qualifies intra-file edges to "src/auth.ts:name" and marks unknown callees as "?:name"
  </behavior>
  <action>
    1. Create `src/parser/calls.ts`:
       - Export `CallEdge` type: `{ caller: string; callee: string }` (bare names from AST)
       - Export `QualifiedCallEdge` type: `{ caller: string; callee: string }` (fully qualified: "file:symbol")
       - Export `extractCallEdges(parser: Parser, language: Parser.Language, source: string, symbols: SymbolInfo[]): CallEdge[]`
       - Export `qualifyEdges(edges: CallEdge[], filePath: string, symbols: SymbolInfo[]): QualifiedCallEdge[]`

    2. `extractCallEdges` implementation:
       - Define TS_CALL_QUERY: `(call_expression function: [(identifier) @callee (member_expression property: (property_identifier) @callee)]) @call`
       - Define PY_CALL_QUERY: `(call function: [(identifier) @callee (attribute attribute: (identifier) @callee)]) @call`
       - Detect language: check if the language query string works for each (or accept a `languageId: "typescript" | "python"` parameter to select the right query)
       - Better approach: accept `languageId` parameter and select query string accordingly
       - Parse source, run query, iterate matches
       - For each call match, find the callee capture text
       - Walk parent chain from the call node upward to find containing function boundary
       - FUNCTION_BOUNDARY_TYPES for TS: "function_declaration", "arrow_function", "method_definition"
       - FUNCTION_BOUNDARY_TYPES for PY: "function_definition"
       - For the containing function node, get its name:
         - function_declaration/function_definition: `.childForFieldName("name")?.text`
         - method_definition: `.childForFieldName("name")?.text`
         - arrow_function: walk up to parent `variable_declarator` and get its `name` field `.childForFieldName("name")?.text`
       - If no containing function found (module-level call), skip this edge
       - Push { caller: callerName, callee: calleeText }
       - IMPORTANT: Do NOT await between setLanguage and parse (not thread-safe)

    3. `qualifyEdges` implementation:
       - Build a Set of known symbol names from the symbols array
       - For each edge:
         - caller is always in this file: qualify as `filePath:callerName`
         - callee: if name exists in symbols Set, qualify as `filePath:calleeName`; otherwise qualify as `?:calleeName`
       - Return QualifiedCallEdge[]

    4. Write tests RED first, then GREEN. Tests use the shared ParserService from loader (same beforeAll pattern as symbols.test.ts).
  </action>
  <verify>
    <automated>bun test src/parser/calls.test.ts --timeout 10000</automated>
  </verify>
  <done>
    - extractCallEdges correctly identifies caller->callee pairs for TS and Python
    - Parent walk correctly identifies the containing function for nested calls
    - Module-level calls produce no edges
    - qualifyEdges correctly produces fully-qualified "file:symbol" paths for intra-file calls and "?:symbol" for cross-file calls
    - All call extraction tests pass
  </done>

<task type="auto" tdd="true">
  <name>Task 2: Wire symbol_deps population and caller warnings into lock.acquire</name>
  <files>src/daemon/rpc/handlers.ts, src/daemon/rpc/handlers.test.ts</files>
  <behavior>
    - lock.acquire on "src/auth.ts:validateToken" reads src/auth.ts from disk, parses it, extracts symbols + call edges, refreshes symbol_deps rows for that file
    - After lock.acquire, symbol_deps table contains qualified call edges for the locked file
    - If callerX calls lockedSymbol, and callerX is locked by another session, lock.acquire response includes warnings: [{ lockedSymbol: "src/auth.ts:callerX", heldBy: otherSessionId, chain: ["src/auth.ts:callerX", "src/auth.ts:lockedSymbol"] }]
    - If no callers are locked by other sessions, warnings array is empty
    - If file does not exist on disk (e.g., symbol path points to deleted file), lock still succeeds but warnings are empty and no symbol_deps are written
    - Warnings are informational -- lock.acquire still returns success (not error)
    - symbol_deps for a file are fully replaced on each parse (delete old rows for that file, insert new)
  </behavior>
  <action>
    1. Add a `parseFileAndRefreshDeps` helper function in handlers.ts (or a separate module if handlers.ts is getting long):
       - Takes: db, parserService, symbolPath (e.g., "src/auth.ts:validateToken")
       - Extract filePath from symbolPath (split on ":", take first part)
       - Determine language from file extension: .ts/.tsx/.js/.jsx -> typescript, .py -> python. If unknown extension, return early (no parsing).
       - Read file from disk: `await Bun.file(resolvedPath).text()`. If file doesn't exist, return early.
       - Resolve file path relative to repo root (WIT_REPO_ROOT env var or cwd)
       - Set language on parser, parse source
       - Extract symbols with extractSymbols
       - Extract call edges with extractCallEdges
       - Qualify edges with qualifyEdges
       - Delete all existing symbol_deps where file = filePath: `db.delete(symbolDeps).where(eq(symbolDeps.file, filePath)).run()`
       - Insert new qualified edges: `db.insert(symbolDeps).values(edges.map(e => ({ file: filePath, caller: e.caller, callee: e.callee }))).run()` (skip if empty)

    2. Add a `buildCallerWarnings` helper function:
       - Takes: db, symbolPath, sessionId (the acquirer's session)
       - Query symbol_deps where callee = symbolPath to find all callers of the locked symbol
       - For each caller, check if that caller's symbol path has an active lock held by a DIFFERENT session
       - Query: join symbolDeps callee=symbolPath with locks on symbolPath=symbolDeps.caller where locks.sessionId != acquirer and locks.expiresAt > now
       - Build warnings array: `{ lockedSymbol: callerSymbolPath, heldBy: lockHolderSessionId, chain: [callerSymbolPath, symbolPath] }`
       - Return warnings array

    3. Wire into lock.acquire handler:
       - After successful lock upsert (before return), call `parseFileAndRefreshDeps`
       - Then call `buildCallerWarnings`
       - Include warnings in the success response: `{ symbolPath, sessionId, acquiredAt, expiresAt, warnings }`

    4. Tests:
       - Create a temp directory with a real .ts file containing functions that call each other
       - Set WIT_REPO_ROOT env to the temp dir in test setup
       - Acquire lock on a callee symbol, verify symbol_deps rows are populated
       - Set up scenario: session-A locks "file.ts:callerFn", then session-B locks "file.ts:calleeFn" where callerFn calls calleeFn -> session-B's response should include warning about session-A's lock on callerFn
       - Test no-warning case: same session holds both locks -> no warning
       - Test file-not-found case: lock.acquire on "nonexistent.ts:foo" succeeds with empty warnings
  </action>
  <verify>
    <automated>bun test src/daemon/rpc/handlers.test.ts --timeout 10000</automated>
  </verify>
  <done>
    - lock.acquire parses the target file, extracts symbols and call edges, and refreshes symbol_deps
    - Caller warnings are returned when callers of the locked symbol are locked by other sessions
    - Warnings are informational (lock still succeeds)
    - symbol_deps rows are fully refreshed per file on each parse (no stale edges)
    - All handler tests pass including new caller warning tests
  </done>
