Methods and their params/results (from src/daemon/rpc/handlers.ts):

1. ping — params: none, result: "pong"
2. register — params: {name: string, sessionId: string}, result: {agentId: number}
3. lock.acquire — params: {symbolPath: string (contains ':'), sessionId: string, ttlMs?: number}, result: {symbolPath, sessionId, acquiredAt: ISO string, expiresAt: ISO string, warnings: CallerWarning[]}
4. lock.release — params: {symbolPath: string, sessionId: string}, result: {released: boolean}
5. lock.query — params: {sessionId?: string}, result: Array<{symbolPath, sessionId, acquiredAt, expiresAt, ttlRemainingMs}>
6. intent.declare — params: {sessionId: string, description: string, files: string[] (min 1), symbols?: string[]}, result: {intentId: string, conflicts: ConflictReport}
7. intent.update — params: {intentId: string, sessionId: string, status: "active"|"resolved"|"abandoned"}, result: {intentId, status, updatedAt: number}
8. intent.query — params: {sessionId?: string, file?: string, status?: string}, result: Array<{intentId, sessionId, description, files, symbols, startByte, endByte, status, declaredAt, updatedAt}>
9. contract.propose — params: {sessionId: string, symbolPath: string (contains ':')}, result: {contractId, symbolPath, signature}
10. contract.respond — params: {contractId: string, sessionId: string, accept: boolean}, result: {contractId, status}
11. contract.query — params: {symbolPath?: string, status?: string}, result: Array<{contractId, proposerSessionId, symbolPath, signature, status, responderSessionId, proposedAt, respondedAt}>
12. check-contracts — params: {files: Array<{path: string, content: string}>}, result: {violations: Array<{contractId, symbolPath, expected, actual}>}

Error codes:
- -32600: INVALID_REQUEST (Zod validation failure)
- -32601: METHOD_NOT_FOUND
- -32000: Application errors (LOCK_CONFLICT, LOCK_NOT_FOUND, LOCK_NOT_HELD, INTENT_NOT_FOUND, INTENT_NOT_OWNED, INVALID_TRANSITION, SYMBOL_NOT_FOUND, CONTRACT_NOT_FOUND, CONTRACT_ALREADY_RESOLVED, SELF_ACCEPT_NOT_ALLOWED)

Protocol envelope:
- Every request: {jsonrpc: "2.0", witVersion: "1", id: string (UUID), method: string, params: object}
- Every success: {jsonrpc: "2.0", witVersion: "1", id: string, result: T}
- Every error: {jsonrpc: "2.0", witVersion: "1", id: string|null, error: {code: number, message: string, data?: unknown}}

Transport: Unix domain socket at .wit/daemon.sock, HTTP POST to /rpc

  <name>Task 2: Write openrpc.json machine-readable specification</name>
  <files>docs/openrpc.json</files>
  <action>
    Create docs/openrpc.json as a valid OpenRPC 1.4.0 document. Structure:

    ```json
    {
      "openrpc": "1.4.0",
      "info": {
        "title": "Wit Agent Coordination Protocol",
        "version": "1",
        "description": "JSON-RPC 2.0 protocol for coordinating multiple AI agents working on the same codebase. Transported over Unix domain socket at .wit/daemon.sock."
      },
      "methods": [...]
    }
    ```

    For each of the 12 methods, create a method entry with:
    - `name`: method name (e.g., "lock.acquire")
    - `summary`: one-line description
    - `params`: array of parameter schema objects. Each param has `name`, `required`, and `schema` (JSON Schema).
    - `result`: result schema object with `name` and `schema` (JSON Schema)
    - `errors`: array of error objects with `code` and `message`

    Use `$defs` (or inline) for shared schemas:
    - ConflictReport schema
    - ConflictItem schema (oneOf with discriminator on "type")
    - CallerWarning schema

    All JSON Schema types must match the actual Zod schemas in handlers.ts exactly:
    - symbolPath fields: `{"type": "string", "pattern": ".*:.*"}` (must contain colon)
    - sessionId: `{"type": "string", "minLength": 1}`
    - ttlMs: `{"type": "integer"}` (optional)
    - files array: `{"type": "array", "items": {"type": "string"}, "minItems": 1}`
    - status enum: `{"type": "string", "enum": ["active", "resolved", "abandoned"]}`

    Validate the JSON is syntactically correct (no trailing commas, proper nesting). The file should be parseable by any JSON parser.
